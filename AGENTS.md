## Language

- すべてのアウトプットは日本語で書くこと。
    - PR タイトル / PR 本文 / 変更概要 / レビューコメント
- 例外: コミットメッセージ
- 例外: コード内の識別子（変数名・関数名・ファイル名）、既存の英語の固有名詞、外部仕様に合わせる必要がある文字列はそのまま。
- 例外: 既存ファイルが英語で統一されているドキュメントを編集する場合は、そのファイルの言語に合わせる。

## PR Template

PR の本文は必ず次の形式（日本語）で書くこと:

## 変更概要

- ...

## 背景 / 目的

- ...

## 変更内容

- ...

## 影響範囲

- ...

## 動作確認

- ...

## 補足

- ...

## リポジトリ構成（monorepo）

- `packages/*`: OSS として公開しているパッケージ群
- `modules/*`: OSS 向けの共通コード
- `dry-run/*`: OSS として公開している eslint-{config,plugin} のテスト稼働用

## MUST（必須・例外なし）

1. パッケージマネージャーは **pnpm のみ** とする。
    - npm / yarn は使用しない。
    - すべての install / 実行コマンドは `pnpm` を用いる。
2. **コードに変更を加えた場合** 、必ず以下を満たした状態で作業を完了すること。
    - `pnpm check-code` が成功していること。
3. **テストを追加または変更した場合** 、以下を満たすこと。
    - テストカバレッジは **75%以上** であること。
    - カバレッジ確認は `pnpm test` **により行う** 。
    - 75% を下回る場合は、テストを追加・修正して基準を満たすこと。
4. コミットメッセージは **Semantic commit messages** に準拠すること
    - Semantic commit messages: https://gist.github.com/joshbuchea/6f47e86d2510bce28f8e7f42ae84c716

## WANT（推奨）

- オブジェクトのプロパティにアクセスする場合は、可能な限り **分割代入** を用いる。
    - 例： `const { foo, bar } = obj;`
    - 同一オブジェクトへのドットアクセスを繰り返す書き方は避ける（合理的な場合を除く）。

## 作業開始時

- 作業ブランチは `develop` から切ること。

## 作業完了条件

- 変更を確定する前に、以下を確認すること。
    - `pnpm check-code` を実行し、成功していること。
- テストを追加・変更した場合：
    - テストカバレッジが 75%以上であることを確認すること。
    - 可能であれば、カバレッジ結果の要約を提示すること。

## 事前相談が必要なケース

以下に該当する場合は、 **作業を進める前に方針を提示し、確認を取ること** 。

- 大規模なリファクタリング
- 依存関係の追加・変更
- テスト構成や設定ファイル（lint / tsconfig / テストランナー等）の変更
